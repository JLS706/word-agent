## 使用流程

1. **先索引**：用 `index_document` 为文档建立向量索引（自动缓存，不会重复索引）
2. **再搜索**：用 `search_document` 进行语义搜索，返回最相关的段落
3. **基于原文回答**：根据搜索到的段落内容回答用户问题，减少幻觉

## 典型问题

- "论文里 MIMO 技术用在了哪里？"
- "第三章讲了什么？"
- "作者对 XXX 方法的评价是什么？"
- "帮我找到关于实验结果的部分"

## 注意

- 需要先索引才能搜索，如果用户直接问内容相关问题，先自动索引
- 搜索返回 Top-3 段落及相关度百分比
- 索引结果会缓存到 `cache/` 目录，同一文档不重复调 API
