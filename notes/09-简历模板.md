# 简历模板 — STAR 法则

> **STAR 法则**：Situation（背景）→ Task（任务）→ Action（行动）→ Result（结果）
>
> 以下提供 2 个简历版本，根据应聘岗位选择。

---

## 版本 A：AI 工程师 / LLM 应用方向

### 项目名称
**DocMaster Agent — 基于 Multi-Agent 架构的学术论文排版智能助手**

### 项目描述（控制在 3 行内）
基于 ReAct (Reasoning + Acting) 架构，从零设计并实现的轻量级 AI Agent 系统。通过自然语言指令驱动 Word 文档自动化处理，支持参考文献格式化、交叉引用生成、格式诊断等 10+ 工具。实现了 Multi-Agent 三角色协作（Planner→Executor→Reviewer）、三级分层记忆、语义切块 RAG、Skill 热插拔、Checkpoint 断点续传等企业级特性。

### 核心贡献（STAR 法则，每条 2-3 行）

**1. 设计并实现 ReAct 推理引擎**
- **S/T**：需要让 LLM 自主规划并执行多步文档操作任务，且能处理执行中的不确定性
- **A**：手写 Think→Act→Observe 循环（~100 行），集成 OpenAI Function Calling 实现工具自主调用；设计三级错误分类（临时/可修正/致命）+ 结构化 Observation 引导 LLM 自修正
- **R**：Agent 首轮工具调用成功率提升至 ~90%，错误自修正覆盖率 >70%

**2. 构建 Multi-Agent 三角色协作流水线**
- **S/T**：单 Agent 处理复杂任务（全面排版一篇论文）时，计划与执行混杂、缺乏验证
- **A**：设计 Planner→Executor→Reviewer Pipeline；实现逐步执行 + 两级验证（规则匹配 + LLM mini-review）+ 三级回溯修正（重试→重规划→跳过）；引入 Checkpoint 断点续传保障长任务可靠性
- **R**：Pipeline 全流程自动化完成率 >85%，中途崩溃可从断点恢复

**3. 设计三级分层记忆系统**
- **S/T**：Agent 需要跨会话学习经验，同时保证核心规则不被 LLM 违反
- **A**：设计 L1 核心规则（铁律，三明治注入 + 后校验）/ L2 长期反思（向量 RAG，冲突融合）/ L3 短期对话（30 天 TTL + 召回率淘汰）三层架构；实现记忆晋升（L3→L2）和效用衰减机制
- **R**：L1 规则服从率 >95%，L2 经验召回准确率 >80%

**4. 实现 Prompt Cache 优化，降低 API 成本 50%+**
- **S/T**：System Prompt 包含动态内容（技能、记忆），导致厂商 KV Cache 永远命中不了
- **A**：将 System Prompt 拆为静态根（L1+工具+行为准则，跨轮次不变）+ 动态叶（技能+记忆，独立消息），确保前缀不变以命中 Prompt Cache；配合动态宪法挂载精选 2-3 条相关规则注入
- **R**：输入 Token 成本降低 50%~90%（取决于厂商 Cache 策略）

**5. 构建四层纵深防御安全沙盒**
- **S/T**：Agent 具备代码执行能力（execute_python），需防止 LLM 生成的代码造成安全风险
- **A**：设计 AST 静态分析 → builtins 白名单 → 进程隔离+超时强杀 → Docker 容器隔离四层防护；实现严格模式（只读分析）和工具模式（允许文件操作）两种安全级别
- **R**：零安全事故，支持 Agent 在沙盒中自主创建和测试新工具

### 技术栈
Python 3.10+ / OpenAI Function Calling / ReAct Agent / Multi-Agent Pipeline / RAG / Embedding / NumPy / FastAPI / Docker / pywin32 (Word COM)

## 版本 B：后端工程师 / 基础架构方向

### 项目名称
**DocMaster Agent — 多智能体文档处理系统**

### 项目描述
设计并实现多智能体协作系统，整合 LLM 推理引擎、插件化工具框架、分层记忆系统、安全沙盒等模块。支持通过 RESTful API 接收自然语言指令，自动化处理 Word 文档。

### 核心贡献

**1. 设计插件化工具框架**
- **S/T**：系统需支持 10+ 工具且允许运行时动态扩展
- **A**：实现 Tool 抽象基类 + ToolRegistry 注册表模式；支持 Agent 运行时创建新工具（代码生成→沙盒验证→用户审批→热注册）；Skill 插件系统支持零代码新增业务（.md 文件热加载）
- **R**：新增工具只需继承基类 + 实现 execute()，零改 Agent 核心代码

**2. 实现状态机驱动的工作流引擎**
- **S/T**：多步骤文档处理流水线需要支持错误恢复和断点续传
- **A**：设计 9 阶段状态机（WorkflowPhase）；实现三级回溯策略（重试→重规划→跳过）；JSON 文件 Checkpoint 持久化，崩溃后可从任意节点恢复
- **R**：流水线中断恢复成功率 100%

**3. 构建向量检索系统（自研，零外部依赖）**
- **S/T**：需要语义检索能力，但不想引入 Pinecone / Milvus 等重型依赖
- **A**：基于 NumPy 实现余弦相似度计算 + VectorStore（JSON 缓存持久化）；设计语义切块算法（Embedding 相似度断崖检测，自适应阈值 μ-1σ）
- **R**：检索延迟 <100ms（千条级），零外部数据库依赖

**4. 设计多层安全架构**
- **S/T**：系统具备代码执行能力，需要纵深防御
- **A**：4 层安全架构（AST→builtins→进程隔离→Docker）；COM 进程守护（PID 精准狙击 + 残留清理）；文档事务（备份→执行→回滚/提交）
- **R**：零安全事故、零进程泄漏

### 技术栈
Python / FastAPI / Docker / Docker Compose / NumPy / pywin32 / OpenAI SDK / JSON 持久化

## 面试口述模板（3 分钟版）

> 面试官问："介绍一下你的项目？"

**参考话术**（根据自己风格调整）：

"这是我独立开发的一个 AI Agent 项目，解决学术论文排版的痛点。

**What**: 用户通过自然语言指令（比如'帮我格式化参考文献并生成交叉引用'），Agent 自动调用 Word COM 接口完成文档操作。

**How**: 底层是我手写的 ReAct 推理引擎——Think-Act-Observe 循环，集成 OpenAI Function Calling 实现工具自主调用。对于复杂任务，我设计了 Multi-Agent Pipeline：Planner 分析制定计划、Executor 逐步执行、Reviewer 验证结果。执行过程中有三级回溯修正——失败了先重试，重试不行就让 Planner 从失败点重新规划，实在不行就跳过标记人工处理。

**特色**: 三个我比较得意的设计——
1. **三级分层记忆**：L1 铁律永不违反、L2 经验向量化 RAG、L3 短期对话自动淘汰，还有从 L3 晋升 L2 的机制
2. **Prompt Cache 优化**：把 System Prompt 拆成静态根和动态叶，命中厂商 KV Cache 降本 50%+
3. **工具自治**：Agent 发现缺功能时可以自己写代码创建新工具，经沙盒测试和用户审批后永久注册

**Result**: 目前已经实际用于处理我自己的毕业论文，Pipeline 全流程自动化完成率 85% 以上。"
